# Authentication - Quick Reference

## TL;DR (30 seconds)

- Use JWT tokens for API authentication
- Tokens expire after 1 hour
- Refresh tokens stored in HTTP-only cookies
- Always validate tokens on server side

---

## Quick Reference (5 minutes)

### Common Tasks

#### Authenticate User

```typescript
const token = await authService.login(email, password);
res.cookie('refreshToken', token.refresh, { httpOnly: true });
return { accessToken: token.access };
```

**When:** User logs in
**File:** `src/services/auth.ts:45`

#### Validate Token

```typescript
const payload = await authService.verifyToken(token);
if (!payload) throw new UnauthorizedError();
```

**When:** Protected route access
**File:** `src/middleware/auth.ts:12`

#### Refresh Token

```typescript
const newTokens = await authService.refresh(refreshToken);
```

**When:** Access token expired
**File:** `src/services/auth.ts:78`

---

## Gotchas

**Avoid:** Storing tokens in localStorage
**Instead:** Use HTTP-only cookies for refresh tokens
**Why:** XSS vulnerability prevention

**Avoid:** Long-lived access tokens
**Instead:** Short access (15min) + refresh tokens
**Why:** Reduces impact of token theft

---

## Validation Checklist

- [ ] Token contains required claims (sub, exp, iat)
- [ ] Token signature is valid
- [ ] Token is not expired
- [ ] User still exists and is active
