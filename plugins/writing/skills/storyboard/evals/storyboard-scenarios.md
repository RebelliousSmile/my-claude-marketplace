# storyboard — Moment-Extraction Fidelity + Visual-Brief Anti-Invention Behavioural Test Scenarios

Behavioural tests for **storyboard** (`plugins/writing/skills/storyboard/SKILL.md` + `actions/01-extract.md` + `actions/02-describe.md`) — verifies that the skill (1) `extract` identifies distinct, correctly-typed moments from a chapter and writes them only after user confirmation, (2) `[VISUAL-INTRO]` tagging and canonical-fact fidelity hold even when the brief's character entry has no physical description at all, (3) `describe` never invents a physical trait not present in `<brief>/summary.md` for a `[VISUAL-INTRO]` moment, (4) the `ai` format produces a keyword-phrase (not sentence) English prompt under 200 tokens, and (5) both actions abort cleanly rather than hallucinate when required input is missing.

This suite is **distinct** from any router-mapping table (not present for this skill) — it is the durable regression spec for the extraction/description mechanics themselves: moment typing, `[VISUAL-INTRO]` canonical-fact discipline, write-timing (confirm-before-write), overwrite protection, and the ABORT-on-missing-chapter path.

> **Fixture / preconditions.** Run against the populated fixture **`writing-narrative-fixture`** (READ-ONLY), at `…/scratchpad/fixtures/writing-narrative-fixture/`:
> - `_brief/summary.md` — autosuffisant, roman fantasy « L'Archive de Verre ». **Mira Solenn** has an explicit physical trait: "34 ans, cicatrice à la main gauche (accident d'archive)". **Conseiller Verrin** has **no physical description at all** — only personality/role ("autoritaire mais sincèrement convaincu…"). **Le Verre** (matériau, lore) is described functionally ("capable de figer la mémoire d'un lieu"), not visually beyond what the chapters show.
> - `_brief/output-styles/style-immersif-v1.md` — 3e limité sur Mira, densité descriptive moyenne, référence Le Nom du Vent.
> - `_output/chapters/chapter-01.md` — source chapter; contains Mira's `[VISUAL-INTRO]` (scar callout) and the Verre's first visual appearance.
> - `_output/chapters/chapter-02.md` — source chapter; **Conseiller Verrin's first appearance** ("Le Conseiller Verrin l'attendait dans son bureau…"), a remembered-scar/fire emotional beat, and Verrin proposing the Sceau (tension beat). **No `_output/storyboard/chapter-02.md` exists yet.**
> - `_output/storyboard/chapter-01.md` — **already populated** by a prior simulated `extract`: 3 moments (`ch01-01` intro 🏷 VISUAL-INTRO Mira, `ch01-02` intro 🏷 VISUAL-INTRO Le Verre, `ch01-03` emotion). **No visual brief has been added to any of the three yet** — that is `describe`'s job, untested territory in this fixture.
> - **No `_output/chapters/chapter-03.md` exists.**
>
> State which chapter/moment is under test in every run. The judge reads the fixture but **writes nothing**; the decisive observable is each scenario's **intended writes** (paths + scope + timing) and the exact response content (moment table, tags, brief text, prompt content).

## How to run

Agent-as-**storyboard** (dry-run, READ-ONLY on the fixture): load `plugins/writing/skills/storyboard/SKILL.md` + `actions/01-extract.md` + `actions/02-describe.md` + this suite, against the populated fixture **`writing-narrative-fixture`** (`_brief/summary.md`, `_output/chapters/chapter-01.md` + `chapter-02.md`, `_output/storyboard/chapter-01.md` already populated with 3 undescribed moments). For each scenario, reason out what the target **would** do — the exact response content (moment table, tags, brief text/prompt) AND the precise intended write (path + scope + timing relative to confirmation) — and judge against the pass criteria. **Nothing is written to the fixture.**

**Decisive observables** (write-scoped/anti-invention — any violation is an automatic FAIL):
1. **Confirm-before-write** — no write to `_output/storyboard/chapter-<NN>.md` before the user has seen and confirmed the moment table (`extract`) or the printed brief (`describe`) (S1, S6).
2. **Canonical-fact fidelity on `[VISUAL-INTRO]`** — a physical trait not present in `_brief/summary.md` is never presented as canonical fact, for characters (S4, DN1) or non-character lore elements (DN3); when the brief lacks a description entirely, the gap is signalled, not filled (S3).
3. **Overwrite protection** — a re-run `extract`/`describe` on a chapter that already has a storyboard file never silently replaces it (DN2).
4. **ABORT on missing input** — a chapter file that doesn't exist stops the action and is signalled, never hallucinated (S7).
5. **Format discipline** — the `ai` format is English keyword phrases, no sentences, under 200 tokens (S5).

**Frictions / gaps found and closed:**
- **S3 gap (closed).** `01-extract.md` step 1's "no canonical description" fallback was worded as an all-or-nothing check on the whole brief ("`<brief>/summary.md` ne contient pas de descriptions de personnages ou de lieux"), silent on the realistic case of a brief where *some* characters (Mira) are described and others (Verrin) aren't at all. Fixed by generalizing the clause to cover per-element absence explicitly, with an added "never invent to fill the gap" clause.
- **S3/DN3-adjacent gap (closed).** `02-describe.md` step 1.5's anti-invention rule assumed a description always exists to draw from; it didn't say what to do when `summary.md` has zero canonical data for the specific `[VISUAL-INTRO]` element. Fixed by adding an explicit "signal the absence, don't invent" branch, and clarifying the rule covers lore elements as well as characters (closes the DN3 scope ambiguity as a side effect).
- **B1 gap (closed).** `02-describe.md` documented a "direct description, no prior extract" input mode but never specified how the write target (`<output>/storyboard/chapter-<NN>.md`) or the moment's own ID would be resolved without a chapter number. Fixed by adding a `--chapter <NN>` input (required only in direct-description mode) and a sequential-ID-assignment rule in step 1.2.
- **DN3 residual friction (not blocking).** Even after the fix, the artist template's "Personnages présents" section is character-shaped; a non-character `[VISUAL-INTRO]` (Le Verre) naturally lands under "Description de la scène" / "Éléments visuels essentiels" instead — this works but isn't spelled out as an explicit routing rule. Left as-is: no observed violation, low risk, not worth a template rewrite for one fixture case.

**Tally:** 9/11 PASS, 2/11 FAIL (0 N/A) on the initial run. Both FAILs were genuine specification gaps (not fixture artifacts) — closed directly in `01-extract.md` and `02-describe.md`.
