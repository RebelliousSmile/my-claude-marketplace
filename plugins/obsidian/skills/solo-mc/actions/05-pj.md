# 05 - PJ

Display the player character sheet with their current in-session state.

## Inputs

- `campaign` (optional, default: from `.current-session`) — string, campaign folder name
- `detail` (optional, default: `compact`) — string, display level: `compact` (tables) or `full` (detailed sections)

## Outputs

Character sheet view:
- Name, archetype, and game system
- Active tags and statuses
- Progress bars and resources
- Key NPC relations
- Active challenges

## Process

1. Detect campaign from `.current-session` if not provided; error if absent.
2. Read `<campaign>/config.yaml` for the game system and default character name.
3. Read `<campaign>/pj/<character>.md` for the base character sheet.
4. Read `<vault>/<jeu>/_campagnes/<campagne>/.session-state.yaml` for current tags, statuses, relations, and countdowns.
5. Render `compact` view (tables per section) or `full` view (detailed narrative sections) according to `detail`.

## Test

Output contains the character name, at least one active tag or status, and a progress indicator.
