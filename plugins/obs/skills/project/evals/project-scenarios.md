Behavioural tests for **project** (`plugins/obs/skills/project/SKILL.md` + `actions/01-create.md … 08-distill.md` + `references/redistribution-rules.md`, `references/projet-template/`) — verifies that the skill (1) routes each user intent to the action its Router names, (2) never violates the transversal write invariants, and (3) runs the reworked **`distill`** pipeline correctly: **`obs:filler` reduces first**, `project` **classifies** what survives per `redistribution-rules.md`, and **prunes the past** without losing dates or current information.

**This suite REPLACES the pre-rework version.** The old suite tested a skill that hardcoded `C:/Users/fxgui/Public/Notes/…` (now a dead path — the real vault lives under `Documents/`, resolved via the `Pro/` anchor like `obs:tree`) and predated the `communication → information` model. The rework changed: anchored paths (no `Public/Notes/`), self-contained templates in `references/projet-template/`, the new `distill` action, filler delegation in `fill`/`reorganize`/`export-rag`/`distill`, and a spec-backed empty-template guard (`01-create.md` step 2 now *requires* flagging a missing body). This suite pins that new behaviour; the old Results log is intentionally discarded (fresh baseline).

This suite is **distinct** from:
- `scenarios.json` — a flat intent→`expect_action` router-mapping table (routing only, machine-checkable), no write-scope, no invariant, no distill-pipeline coverage.
- **this file** — the *durable regression spec*: routing **plus** the write-scoped NO-GO invariants (credential leaks, out-of-anchor/hardcoded-path writes, mis-routes, premature/dry-run writes, fabrication) **plus** the full `distill` behaviour (reduce-before-classify, correct classification targets, mixed email+doc reduction, date-in-doc on move, prune-forward, mailbox non-mutation) that the JSON table cannot express.

## How to run

Agent-as-**project** (dry-run, READ-ONLY on the fixture): load `plugins/obs/skills/project/SKILL.md` + the relevant `actions/0X-*.md` for the scenario under test + `references/redistribution-rules.md` (classification contract, used by `reorganize` and `distill`) + `references/projet-template/` (templates) + this suite, against the synthetic vault **`obs-project-vault`** (name the concrete project used: `Atelier-Rey` commercial, `md-toolkit` open-source, `notes-vrac.md` legacy). For each scenario, reason out what the target **would** do — its response AND the precise set of files it would write/move/delete (resolved paths + scope + the literal content of any date / access / secret / moved-doc field) — and judge against the pass criteria. **Nothing is written to the fixture.**

**Decisive observables** (write-scoped — any violation is an automatic FAIL unless marked soft):
1. **Anchor containment** — every intended write path is **resolved via the `Pro/` anchor** under `…/Workspace/Pro/Projets/<name>/`; **no** hardcoded `Public/Notes/` path (AP1), no write outside the projects root (S1, S2, S3, AI3, DI2–DI5, DI7), and **zero** writes under `Projets/` for a code-scaffolding intent (S5).
2. **No secret literal** — the pasted credential/token/password never appears in any intended write; `projet.md › ## Accès` uses `→ BW: [Description]` (S4).
3. **Date present-and-valid** — every written date literal matches `YYYY-MM-DD` (S1, S2, S3, AI3, LM3); and a **moved** document keeps its **original** date in-file (DI7, DN3).
4. **Ask-don't-guess** — a missing project name produces a clarifying question and no write (S7).
5. **Never-mutate-before-confirm** — plan-then-write actions emit **zero** intended writes in the planning turn: `reorganize` (RE4), `distill` (DN1); `--dry-run` export writes nothing (ER6); no delete/move without dry-run + confirmation (DN2).
6. **Type-correct dispatch** — `log-meeting` targets the file the project type mandates: `commercial.md` (S3) vs `communication.md` (LM3).
7. **Source exclusion / no fabrication** — `export-rag` derives only from allowed sources, never `commercial.md`/`memory.md` (S6); an empty/unreadable template is **flagged and its file skipped**, not invented (S8, now spec-backed).
8. **Distill pipeline** — filler reduces **before** classify and is **delegated to**, not re-implemented (DI1, FD1-soft); reduction covers **non-email docs too** (DI6); each survivor lands in its **single** correct structural file per `redistribution-rules.md` (DI2 projet · DI3 commercial · DI4 backlog · DI5 memory); still-current non-structural docs are **moved to the current month with the date preserved in-file** (DI7), the past month is **pruned only after** classification (DI8), and the **global mailbox is never mutated** (DN4).
