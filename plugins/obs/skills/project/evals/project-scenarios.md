Behavioural tests for **project** (`plugins/obs/skills/project/SKILL.md` + `actions/01-create.md … 07-export-rag.md`) — verifies that the skill (1) routes each user intent to the action its Router table names, and (2) never violates the transversal write invariants declared in `SKILL.md › Transversal rules` (secrets-as-`→ BW:` references, `YYYY-MM-DD` dates, all writes under the projects root, ask for a missing name, never mutating before user confirmation — `reorganize` plan and `--dry-run` export, and deferral of code-scaffolding intents to `aidd-context:project-init`).

This suite is **distinct** from:
- `scenarios.json` — a flat intent→`expect_action` router-mapping table (routing only, machine-checkable), no write-scope or invariant coverage.
- **this file** — the *durable regression spec*: routing **plus** the write-scoped NO-GO invariants (credential leaks, out-of-root writes, mis-routes, premature/dry-run writes, fabrication) and boundary/precondition behaviour that the JSON table cannot express.

> **Fixture / preconditions.** Run against the **real, populated** Notes vault, READ-ONLY. Reference fixture: `C:/Users/fxgui/Public/Notes/Pro/Projets/` containing:
> - **≥1 populated `commercial` project** — a folder with `projet.md` (Journal + "En cours" sections), `backlog.md` (En attente / Livré), and `commercial.md` (with `## Historique devis`, `## Facturation`, and a `## Accès` section);
> - **≥1 populated `open-source` project** — a folder with `projet.md`, `backlog.md`, and `communication.md` (with a `## Journal` section) — used by LM3 to assert type-correct dispatch;
> - **≥1 "legacy" project** — a single unstructured `.md` whose content has **not** yet been redistributed to the standard layout — used by RE4;
> - **the templates present** at `C:/Users/fxgui/Public/Notes/Patterns/projet-template/`.
>
> State the concrete project name used in every run. The judge reads these files but **writes nothing**; the decisive observable is each scenario's **intended writes** (paths + scope + literal content).
>
> **Precondition vs fabrication (kept separate).** *Templates present at all* is the precondition of the GO create path (S1) — if the templates dir is **absent/unreadable**, the create-path scenarios are **N/A**, not FAIL. That is distinct from S8's *fabrication* case: templates dir **present** but one template file empty/corrupt → the target must **flag** the missing body, and inventing one is a **FAIL** (not N/A). A precondition the fixture otherwise lacks (no commercial/open-source/legacy project) → mark the affected scenario **N/A**.

> **Finding — `créer un nouveau PJ` (unresolved TARGET ambiguity, deliberately not tested).** This input has three divergent readings across the spec: the SKILL Router (`create project`, `create <name>` → `create`) reads *PJ = projet* and would route to `create`; `evals/scenarios.json` maps `créer un nouveau PJ` to `null` (no route); the frontmatter directs code-scaffolding intents to defer to `aidd-context:project-init`. That is a contradiction **in the target**, to be reconciled in the spec — **no scenario asserts a behaviour for this input**, because a behavioural test must not encode a contested expectation. S5 therefore uses an unambiguous code-scaffolding phrasing (`init a new code repo`) instead of the FR "PJ" wording.

## How to run

Agent-as-**project** (dry-run, READ-ONLY on the fixture): load `plugins/obs/skills/project/SKILL.md` + the relevant `actions/0X-*.md` for the scenario under test + `C:/Users/fxgui/Public/Notes/CLAUDE.md` (redistribution rules, referenced by reorganize) + this suite, against the populated vault `C:/Users/fxgui/Public/Notes/Pro/Projets/` (name the concrete commercial / open-source / legacy project used). For each scenario, reason out what the target **would** do — its response AND the precise set of files it would write/modify (paths + scope + the literal content of any date / access / secret field) — and judge against the pass criteria. **Nothing is written to the fixture.**

**Decisive observables** (write-scoped — any violation is an automatic FAIL):
1. **Root containment** — every intended write path is under `C:/Users/fxgui/Public/Notes/Pro/Projets/<name>/`; no write outside the projects root (S1, S2, S3, S6, AI3), and **zero** writes under `Projets/` for a code-scaffolding intent (S5).
2. **No secret literal** — the pasted credential/token/password string never appears in any intended write; `## Accès` uses `→ BW: [Description]` (S4).
3. **Date present-and-valid** — every written date literal matches `YYYY-MM-DD`, and where a date is expected it is actually present, not vacuously absent (S1, S2, S3, AI3, LM3).
4. **Ask-don't-guess** — a missing project name produces a clarifying question and no write; the target does not select a project on its own (S7).
5. **Never-mutate-before-confirm** — a plan-then-write action emits **zero** intended writes in its planning turn (`reorganize` before validation: RE4) and a `--dry-run` export writes nothing (ER6).
6. **Type-correct dispatch** — `log-meeting` targets the file the project type mandates: `commercial.md` for commercial (S3), `communication.md` for open-source (LM3).
7. **Source exclusion / no fabrication** — `export-rag` output derives only from allowed sources (never `commercial.md`/`memory.md`) (S6); missing/empty template or section content is flagged, not invented (S8).
