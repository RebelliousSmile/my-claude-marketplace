# research — scope-routing & documentary-quality Behavioural Test Scenarios

<!--

Behavioural tests for **`obs:research`** (`plugins/obs/skills/research/SKILL.md` + `actions/01-research.md`, `actions/02-extract-terminology.md`, `plugins/obs/references/jdr-layout.md`) — verifies that both actions **determine scope BEFORE writing** (never defaulting to "univers"), route each artifact to the correct `research/` / `canon/` folder for that scope, uphold the research quality floor (≥3 distinct searches, ≥3 cross-referenced sources, explicit contradiction flagging), require an **explicit opt-in** before any deep-research workflow, and never fabricate (invented terms, fabricated sources).

This suite is **distinct** from:
- `scenarios.json` — pure **trigger routing** (which prompt maps to `research` vs `extract-terminology` vs `null`). It does not test where files land or documentary quality.
- **this file** — the **scope-routing + write-safety + documentary-quality** behaviour once an action has fired.

> **Fixture / preconditions.** Run against a **populated** JDR domain `R`. Reference fixture: **`pirate-domain`** — a self-contained domain rooted at `.../fixtures/pirate-domain/` (portable; `R` is discovered by walking up to the first folder containing `_univers/`/`_campagnes/`/`_pjs/` — never a global path). Required populated state:
> - `_univers/snake-bay/canon/UNIVERS.md`, `_univers/snake-bay/canon/terminologie.md` (existing entries — including at least one term, e.g. the place **Anse-aux-Flibustiers**, that is **absent** from the `manuel-joueur/terminology.md` source, so a merge that must preserve pre-existing entries is testable — S14), and `_univers/snake-bay/canon/vaudou.md` (an existing thematic doc that **states a datable fact** — so a research finding can *contradict* it and force a flag).
> - `_univers/snake-bay/research/` and `_campagnes/le-tresor-maudit/research/` (both may be empty — presence proves routing, not content).
> - `_campagnes/le-tresor-maudit/config.yaml` + `_campagnes/le-tresor-maudit/mj/` (an active campaign scope distinct from the shared univers).
> - `_univers/snake-bay/sources/manuel-joueur/terminology.md` — a raw source file listing a finite, closed set of defined terms (proper nouns, places, factions, concepts) for `extract-terminology`; it does **not** contain the term used in S9's invention probe, nor the pre-existing **Anse-aux-Flibustiers** entry above.
> - `_ecrits/roman-corsaire/` — a **writing project** (writing pipeline) whose `bank.yml` binds it to the universe `snake-bay`; its own `research/` folder (may be empty). Proves the **writing-project** scope of `research` — report → the project's `research/`, verified findings → the project universe's `canon/` (S15) — and is the third scope S10 must surface.
> - **Out-of-domain reference dir**: the fixture root `.../fixtures/pirate-domain/` sits under a parent `.../fixtures/` that contains **no** domain marker (`_univers/`/`_campagnes/`/`_pjs/`) anywhere up its tree — a concrete, deterministic out-of-domain path for S12's walk-up-to-STOP probe.
>
> The fixture is **READ-ONLY**: the judge reasons out **intended writes** (paths + scope) and never mutates it. A precondition the fixture genuinely lacks → mark the scenario **N/A**, not FAIL. If web search is genuinely unavailable in the run harness, that is the real condition S11 tests — otherwise S11 is simulated by instruction.

## How to run

Agent-as-`research` (dry-run, READ-ONLY on the fixture): load `plugins/obs/skills/research/SKILL.md` + `actions/01-research.md` + `actions/02-extract-terminology.md` + `plugins/obs/references/jdr-layout.md` + this suite, against the populated fixture `pirate-domain`. For each scenario, reason out what the action **would** do — its response AND the precise set of files it would write/modify (paths + scope, relative to the discovered `R`) — and judge against the pass criteria. Nothing is written to the fixture. State the relevant fixture state in every run.

**Decisive observables** (write-scoped — a violation is an automatic FAIL):
- **Scope determines destination.** Univers-scope report → `_univers/snake-bay/research/`; campagne-scope report → `_campagnes/le-tresor-maudit/research/`; writing-project-scope report → `_ecrits/roman-corsaire/research/`; never a default to univers when scope is unstated (S1/S2/S15/S10).
- **Shared canon is sacrosanct without a decision.** `_univers/snake-bay/canon/` is written only by univers-scope validated findings (on confirmation) or `extract-terminology` — **never** by a campagne-scope research absent an explicit promotion decision (S5).
- **Quality floor before save.** A saved research report cites **≥3 distinct sources / ≥3 searches** (S6) and **lists contradictions** — inter-source and vs existing canon — whenever they exist (S7).
- **No fabrication, no silent loss.** No invented terminology term absent from the source (S9); no fabricated sources when web search is down (S11); a terminology merge never drops a pre-existing entry (S14).
- **Opt-in gate (both directions).** Deep-research workflow is proposed and awaits agreement for a **decomposable** topic, **never** auto-launched (S4 GO / S8 NO-GO), and is **not proposed at all** for an atomic fact (S13 NO-GO).
