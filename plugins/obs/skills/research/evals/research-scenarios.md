# research — scope-routing & documentary-quality Behavioural Test Scenarios

Behavioural tests for **`obs:research`** (`plugins/obs/skills/research/SKILL.md` + `actions/01-research.md`, `actions/02-extract-terminology.md`, **generic model and JDR profile** `plugins/obs/references/domain-layout.md` (its § JDR profile)) — verifies that both actions operate on the **generic domain model** by default and apply the **JDR profile only when detected**, that they **determine scope BEFORE writing** (never defaulting), route each artifact to the correct `research/` + synthesized-knowledge folder **for the detected profile** (`reference/` generic · `canon/` JDR), uphold the research quality floor (≥3 distinct searches, ≥3 cross-referenced sources, explicit contradiction flagging), gate any deep-research workflow behind an **explicit opt-in**, and never fabricate (invented terms, fabricated sources, dropped merge entries).

> **Generic core vs JDR profile.** The skill's baseline is the **generic model** (`domain-layout.md`): a domain `R` with freely-named working-dir buckets, synthesized knowledge under `<bucket>/reference/`, scopes **`shared`** (R-level, durable, reused across work units) vs **`project`** (specific to one work unit). The **JDR profile** (`domain-layout.md` § JDR profile) is applied **conditionally** — only when `R/bank.yml` declares `profile: jdr` **or** `R` contains the signature buckets `_univers/`/`_systeme/`. The profile refines `reference/` into a `canon/`+`mj/` provenance split and **adds a third scope `campagne`** (specific to one game). This suite exercises **both**: a generic fixture (`naval-history`, no profile) where only `shared`/`project` exist and knowledge lands in `reference/`; and a JDR fixture (`pirate-domain`, profile active) where `campagne` is exposed and knowledge lands in `canon/`. **Profile detection is itself under test** (a generic domain must NOT surface `campagne`; a JDR domain must).

This suite is **distinct** from:
- `scenarios.json` — pure **trigger routing** (which prompt maps to `research` vs `extract-terminology` vs `null`). It does not test where files land, profile detection, or documentary quality.
- **this file** — the **profile-aware scope-routing + write-safety + documentary-quality** behaviour once an action has fired.

> **Fixture / preconditions.** Two **populated, READ-ONLY** fixtures; the judge reasons out **intended writes** (paths + scope, relative to the discovered `R`) and never mutates either. A precondition a fixture genuinely lacks → mark the scenario **N/A**, not FAIL. If web search is genuinely unavailable in the run harness, that is the real condition G14 tests — otherwise it is simulated by instruction.
>
> **Fixture A — `naval-history` (GENERIC, no profile).** A self-contained subcategory domain rooted at `.../fixtures/Perso/Redaction/histoire-navale/`; `R = histoire-navale` is discovered by the **`obs:tree` anchor** (walk up to a `Perso`/`Pro` segment — here `Perso` — the subcategory below is `R`). **No JDR profile**: `R/bank.yml` has **no `profile:` key** and `R` contains **no** `_univers/`/`_systeme/` bucket → the **generic core** applies (scopes `shared`/`project`; synthesized knowledge → `reference/`; **no `campagne` scope**). Required populated state:
> - `_kb/` — a freely-named **shared bucket** (the `shared` scope root), catalogued by `bank.yml`, containing:
>   - `_kb/reference/apercu.md` — an existing synthesized doc that **states a datable fact** (e.g. "le chronomètre H4 de Harrison fut éprouvé en 1761"), so a research finding can *contradict* it and force a flag (G10).
>   - `_kb/reference/terminologie.md` — existing terms including at least one (e.g. **Astrolabe de mer**) that is **absent** from the source glossary below, so a merge that must preserve pre-existing entries is testable (G12).
>   - `_kb/sources/glossaire-nav/glossaire.md` — a raw source listing a finite, closed set of defined terms; it does **not** contain G11's invention-probe term, nor the pre-existing **Astrolabe de mer** entry.
>   - `_kb/research/` — may be empty (presence proves routing, not content).
> - `2026/07/le-cap-horn/` — a **work unit (project)** under the domain's dated axis (holds `_brief/`, `research/` (may be empty), `sources/`); it is the `project` scope root. There is **no** `_campagnes/` anywhere in this domain.
> - **Out-of-domain path**: the fixture's ancestor `.../fixtures/` (above the `Perso` segment) contains **no** `Perso`/`Pro` segment anywhere up its tree — a concrete, deterministic out-of-domain path for G13's walk-up-to-STOP probe.
>
> **Fixture B — `pirate-domain` (JDR, profile active).** A self-contained domain rooted at `.../fixtures/pirate-domain/`; `R` is discovered by the JDR marker shortcut (walk up to the first folder containing `_univers/`/`_campagnes/`/`_pjs/`). The profile is active (`_univers/`/`_campagnes/` present, or `profile: jdr` in `bank.yml`) → `canon/`+`mj/` split and the **`campagne`** scope apply. Required populated state:
> - `_univers/snake-bay/canon/UNIVERS.md`, `_univers/snake-bay/canon/terminologie.md`, and `_univers/snake-bay/canon/vaudou.md` (an existing thematic doc that **states a datable fact** — so a finding can contradict it and force a flag, J-floor context).
> - `_univers/snake-bay/research/` and `_campagnes/le-tresor-maudit/research/` (both may be empty — presence proves routing).
> - `_campagnes/le-tresor-maudit/config.yaml` + `_campagnes/le-tresor-maudit/mj/` (an active **campagne** scope distinct from the shared univers).
> - `_univers/snake-bay/sources/manuel-joueur/terminology.md` — a raw source listing a finite, closed set of terms for `extract-terminology`.
> - `_ecrits/roman-corsaire/` — a **writing project** whose `bank.yml` binds it to the universe `snake-bay`; its own `research/` (may be empty). Proves the JDR **writing-project** scope and is the third scope J5 must surface.

## How to run

Agent-as-`research` (dry-run, READ-ONLY on the fixtures): load `plugins/obs/skills/research/SKILL.md` + `actions/01-research.md` + `actions/02-extract-terminology.md` + `plugins/obs/references/domain-layout.md` (generic model + JDR profile in its § JDR profile) + this suite. Run **G-scenarios against Fixture A (`naval-history`, generic — profile OFF)** and **J-scenarios against Fixture B (`pirate-domain`, JDR — profile ON)**. For each scenario, reason out what the action **would** do — its response AND the precise set of files it would write/modify (paths + scope, relative to the discovered `R`) — and judge against the pass criteria. Nothing is written to either fixture. State the relevant fixture + its detected profile in every run.

**Decisive observables** (write-scoped — a violation is an automatic FAIL):
- **Profile detection drives the target folder.** On the **generic** fixture, synthesized knowledge → `reference/` and **only `shared`/`project`** scopes exist — surfacing or writing a `campagne`/`canon/` there is FAIL (G4/G5). On the **JDR** fixture, knowledge → `canon/` and the **`campagne`** scope is exposed (J2/J5). G5↔J5 is the detection discriminant.
- **Scope determines destination, never a default.** shared → `<bucket>/research/`; project → `<work-unit>/research/`; campagne (JDR only) → `<campagne>/research/`; writing project (JDR) → `<projet>/research/` (G1/G2/J1/J2/J6). No default write before scope is resolved (G5/J5).
- **Synthesized-knowledge folder is sacrosanct without a decision.** Generic `reference/` and JDR univers `canon/` are written only by a same-scope validated-findings confirmation or `extract-terminology` — **never** by a campagne-scope research absent an explicit promotion decision (J4).
- **Quality floor before save.** A saved report cites **≥3 distinct sources / ≥3 searches** (G9) and **lists contradictions** — inter-source and vs the existing scope doc — whenever they exist (G10).
- **No fabrication, no silent loss.** No invented terminology term absent from the source (G11); a terminology merge never drops a pre-existing entry (G12); no fabricated sources when web search is down (G14).
- **Opt-in gate (both directions).** Deep-research workflow is proposed and awaits agreement for a **decomposable** topic, **never** auto-launched (G6 GO / G7 NO-GO), and is **not proposed at all** for an atomic fact (G8 NO-GO).
