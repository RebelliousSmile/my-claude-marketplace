## How to run

Agent-as-filler (dry-run, **READ-ONLY** sur le fixture) : charger `SKILL.md` (Philosophie, Default flow, Transversal rules 1–10) + les `actions/` concernées (`01-survey`, `02-sort`, `03-digest`, `04-index`, `05-merge`, `06-clean`, `07-condense`, `08-synthesize`) + `references/email-md-format.md` + `references/digest-matrix.md` + cette suite, contre le fixture peuplé `Perso/Finances/Assurances/2026/06/` (le niveau `<Subcategory>` transversal `Perso/Finances/Assurances/` pour S10, l'état secondaire `~/Downloads/dump-factures/` pour B1, l'état tertiaire `Perso/Finances/Assurances/2025/` pour B2). Pour chaque scénario, raisonner **ce que filler ferait** — sa réponse ET l'ensemble précis des fichiers qu'il écrirait / déplacerait / supprimerait (chemins + scope). **Rien n'est écrit dans le fixture.**

**Observables décisifs** (violation = FAIL automatique) :
1. **`survey` / lecture seule** : intended writes = ∅ pour toute route `survey` (S1, S9, B2).
2. **Sources humaines intouchées par `digest`** : le thread `subject_hash: a1b2c3`, la note RDV et son doublon ne sont jamais supprimés ni transformés en tableau ; `digest` ne touche que le groupe notifications homogène matchant le filtre (S3, S5).
3. **Plan affiché AVANT mutation** : aucune suppression (`clean`) ni déplacement (`sort`) sans plan + confirmation explicite (S2a, S6).
4. **Suffixe numérique vs écrasement** : cible existante → chemin suffixé `_2`, l'original jamais écrasé (S8).
5. **Scope non-récursif** : aucun chemin sous un sous-répertoire (`pieces-jointes/`) touché sans « récursivement » explicite (S7).
6. **Niveau d'artefact correct** : groupe temporel → reste dans `YYYY/MM` (S3) ; transversal → `<Subcategory>` (S10) ; hors tree → `path/_…` (B1).
7. **`merge`/`index` préservent** : 1 artefact `_`-préfixé écrit, 0 source supprimée ou modifiée (S12, S14).
8. **Gate de confirmation `digest`** : aucune écriture ni suppression avant le « oui » final de l'aperçu (S15).
9. **Résolution d'entité `sort`** : réponse du propriétaire bucketée sous le correspondant (`to:`), slugs kebab-case (S2b).
