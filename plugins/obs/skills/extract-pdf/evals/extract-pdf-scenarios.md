# extract-pdf — Fidelity & scope invariants across the generic core AND the JDR profile — Behavioural Test Scenarios

Behavioural tests for **extract-pdf** (`plugins/obs/skills/extract-pdf/SKILL.md` + `actions/01-setup.md`, `02-process-chunk.md`, `03-distribute.md`, `04-debug.md` + `references/domain-layout.md` as the **primary generic** convention + `references/jdr-layout.md` as the **conditional JDR profile**) — verifies that the extraction pipeline stays **faithful to the source PDF** and **scoped to `sources/`** across the `setup → process-chunk (×N) → distribute` routing (with `debug` on demand), **on both layouts of the agnosticized skill**:

- **Generic core** (a non-JDR domain — no `profile:` key, no `_univers/`/`_systeme/` buckets): raw sources land under a **single resolved** `<target>/sources/<source>/`; the skill **never** writes into the generic synthesized layer `reference/`; `R` is located by the `obs:tree` anchor (walk up to a `Perso`/`Pro` segment, subcategory = `R`) with **no domain marker required**.
- **JDR profile** (a game domain — `profile: jdr` in `bank.yml`, or `_univers/`/`_systeme/` present): raw sources **split by provenance** (lore/terminology → `_univers/<u>/sources/`, rules → `_systeme/sources/`); the skill **never** writes into `canon/`/`mj/` (that ventilation is the downstream role of `ttrpg:lore-extract` / `ttrpg:rules-keeper`); `R` is located by the marker shortcut (`_campagnes/`/`_univers/`/`_pjs/`).

The suite is deliberately built on **two fixtures** so the shared behavioural invariants (no invention, `sources/`-only, `fulltext.md` preserved, `pending`/`done`/`failed` vocabulary, local `R` resolution, validation gate, read-only diagnostics) are each exercised on the layout where they bite — and so that **profile detection itself** is pinned: a generic domain must resolve **generic** targets, and applying the JDR universe/system split to a generic domain is an automatic FAIL.

This suite is **distinct** from:
- `evals/scenarios.json` — trigger→action **routing** ("extract chunk" → `process-chunk`, "ventile le lore dans canon/" → `null`). It pins *which action fires*, not *what the action writes*.
- **this file** — the **write-scoped fidelity, scope, and profile-detection invariants** each action must honour once it fires, across the generic core and the JDR profile.

> **Fixture / preconditions.** Two **populated** fixtures, both READ-ONLY. State which one (and its relevant state) in every run. A precondition the fixture lacks (e.g. a chunk still `pending` for a `distribute` GO) → mark the scenario **N/A**, not FAIL. A missing concrete value (unreadable PDF page) → **data limit**, reported separately — the target should flag it, never fabricate.
>

## How to run

Agent-as-extract-pdf (dry-run, READ-ONLY on the fixtures): load `SKILL.md` + `actions/01-setup.md`, `02-process-chunk.md`, `03-distribute.md`, `04-debug.md` + `references/domain-layout.md` (**primary generic**) + `references/jdr-layout.md` (**JDR profile**, via `plugins/obs/references/jdr-layout.md`) + this suite. Run S1–S9 against **Fixture A `climat`** (generic core active), S10–S16 against **Fixture B `engrenages`** (JDR profile active). For each scenario, reason out what the target **would** do — its response AND the precise set of files it would write/modify/delete (paths + scope + ordering) — and judge against the pass criteria. Nothing is written to either fixture.

**Decisive observables** (write-scoped — any violation is an automatic FAIL):
1. **Synthesized layer untouched** — generic: `_corpus/reference/` receives **zero** writes (S3, S5); JDR: `_univers/rouages/canon/`, `_univers/rouages/mj/`, `_systeme/canon/`, `_systeme/mj/` receive **zero** writes (S11, S12, S13).
2. **Correct target by profile** — generic: a **single** `<target-root>/sources/<source>/`, no provenance split (S3, S6); JDR: lore/terminology → `<univers-root>/sources/`, rules → `<systeme-root>/sources/`, nothing crossing over (S11, S12). Applying the JDR split to a generic domain (or the reverse) = FAIL (S6).
3. **Profile detection** — a domain with no `profile:` key and no `_univers/`/`_systeme/` resolves the **generic** table (S1, S3, S6); a domain with `profile: jdr` or those buckets resolves the **JDR** table (S10, S11, S12).
4. **`fulltext.md` present & sequenced** — every populated `sources/<source>/` contains a non-empty `fulltext.md`, assembled **before** any `raw/`/`chunks/`/`classified/` deletion (S3, S11, S12, S14).
5. **Status vocabulary + working path** — `progress.md` Status ∈ {`pending`,`done`,`failed`} and the `Univers` field is present (any `TODO`/`DONE` = FAIL); the working tree is `docs/extraction/`, never `.docs/` (S1, S2, S7, S10).
6. **Source-anchored** — every classified/merged item is traceable to the chunk's `raw/*.txt`; no fabricated content (S2, S4).
7. **Validation gate before classified writes** — no `classified/*.md` write precedes the 500-char preview + `[Y/n]` checkpoint (S9).
8. **Read-only diagnostics** — a `distribute` completeness STOP and a `debug` audit both emit reports with **zero** intended writes (S8, S16).
9. **Local `R` resolution** — generic: `R` via the `obs:tree` `Perso`/`Pro` anchor, no marker required (S1); JDR: `R` via the marker shortcut (S10); an absent anchor/marker yields a report, not a write, and no absolute/global path is emitted (S15).
