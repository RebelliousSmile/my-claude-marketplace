# extract-pdf ‚Äî Fidelity & scope invariants of the extraction pipeline ‚Äî Behavioural Test Scenarios

Behavioural tests for **extract-pdf** (`plugins/obs/skills/extract-pdf/SKILL.md` + `actions/01-setup.md`, `02-process-chunk.md`, `03-distribute.md`, `04-debug.md` + `references/jdr-layout.md`) ‚Äî verifies that the extraction pipeline stays **faithful to the source PDF** and **scoped to `sources/`** across the `setup ‚Üí process-chunk (√óN) ‚Üí distribute` routing (with `debug` on demand): it never invents content, never writes into `canon/`/`mj/`, always preserves the raw `fulltext.md`, uses the normalized `progress.md` status vocabulary, and resolves the game domain `R` locally.

This suite is **distinct** from:
- `evals/scenarios.json` ‚Äî trigger‚Üíaction **routing** ("extract chunk" ‚Üí `process-chunk`, "ventile le lore dans canon/" ‚Üí `null`). It pins *which action fires*, not *what the action writes*.
- **this file** ‚Äî the **write-scoped fidelity & scope invariants** each action must honour once it fires (no invention, `sources/` only, `fulltext.md` preserved, status vocabulary, local `R` resolution, companion = distinct source).

> **Fixture / preconditions.** Run against a **populated** game domain `R`, READ-ONLY. Reference fixture: **`engrenages`** ‚Äî `Perso/RPG/engrenages/` (self-contained `R`, marker `_campagnes/` present), containing:
> - `_univers/rouages/` with `canon/` (populated: `terminologie.md`, `factions.md`), `mj/` (populated), `sources/` (empty) ‚Äî universe root.
> - `_systeme/` with `canon/` (populated), `mj/` (populated), `sources/` (empty) ‚Äî system root.
> - `2026/06/import-corebook/` ‚Äî the writing project directory (`R/<AAAA>/<MM>/<projet>/`), containing:
>   - `docs/extraction/engrenages-regles/` ‚Äî **in-progress** extraction: `progress.md` (`Univers: rouages`, 4 chunks ‚Äî `part01`/`part02` = `done`, `part03`/`part04` = `pending`), `chunks/*_partNN_*.pdf` (4 files), `raw/chunk_01.txt` **empty** (injected inconsistency: `part01` is `done` yet its raw file is empty ‚Äî the debug fixture for S24), `chunk_02.txt` non-empty, `classified/lore.md` + `rules.md` (chunks 1‚Äì2, with YAML markers).
>   - `docs/extraction/engrenages-livret-joueur/` ‚Äî **fully done** companion extraction (player booklet): `progress.md` (all chunks `done`), `chunks/`, `raw/chunk_01.txt`‚Ä¶`chunk_03.txt`, `classified/lore.md` + `terminology.md`.
>   - `docs/extraction/engrenages-mecanismes/` ‚Äî **fully done** rules-only companion extraction (advanced-mechanics booklet): `progress.md` (`Univers: rouages`, all chunks `done`), `chunks/`, `raw/chunk_01.txt`‚Ä¶`chunk_02.txt`, `classified/rules.md` **only** (no lore/terminology ‚Äî a system-side source; the S19 fixture for the `_systeme/` half of the distribution table).
>   - `docs/prompts/workshop/` ‚Äî the four templates (`extract.prompt.md`, `extract-chunk.prompt.md`, `extract-distribute.prompt.md`, `extract-debug.prompt.md`).
>   - Source PDFs on disk: `engrenages-regles.pdf`, `engrenages-livret-joueur.pdf`, `engrenages-mecanismes.pdf`, `engrenages-ecran-mj.pdf` (MC screen ‚Äî no extraction folder yet).
>
> State the relevant fixture state in every run. A precondition the fixture lacks (e.g. a chunk still `pending` for a `distribute` GO) ‚Üí mark the scenario **N/A**, not FAIL. A missing concrete value (unreadable PDF page) ‚Üí **data limit**, reported separately ‚Äî the target should flag it, never fabricate.

## How to run

Agent-as-extract-pdf (dry-run, READ-ONLY on the fixture): load `SKILL.md` + `actions/01-setup.md`, `02-process-chunk.md`, `03-distribute.md`, `04-debug.md` + `references/jdr-layout.md` (via `plugins/obs/references/jdr-layout.md`) + this suite, against the populated fixture **`engrenages`**. For each scenario, reason out what the target **would** do ‚Äî its response AND the precise set of files it would write/modify/delete (paths + scope + ordering) ‚Äî and judge against the pass criteria. Nothing is written to the fixture.

**Decisive observables** (write-scoped ‚Äî any violation is an automatic FAIL):
1. **`canon/` and `mj/` untouched** ‚Äî `_univers/rouages/canon/`, `_univers/rouages/mj/`, `_systeme/canon/`, `_systeme/mj/` receive **zero** intended writes across the whole pipeline (S3, S5, S16, S19).
2. **`fulltext.md` present & sequenced** ‚Äî every populated `sources/<source>/` contains a non-empty `fulltext.md`, assembled **before** any `raw/`/`chunks/`/`classified/` deletion (S3, S6, S19).
3. **Correct destination by category** ‚Äî lore/terminology ‚Üí `<univers-root>/sources/`, rules ‚Üí `<systeme-root>/sources/`; nothing crosses over (S3 univers half, S19 system half).
4. **Status vocabulary + working path** ‚Äî `progress.md` Status column ‚àà {`pending`,`done`,`failed`} and the `Univers` field is present (any `TODO`/`DONE` = FAIL); the working tree is `docs/extraction/`, never `.docs/` (S1, S2, S7).
5. **Source-anchored** ‚Äî every classified/merged item is traceable to the chunk's `raw/*.txt`; no fabricated content (S2, S4).
6. **Validation gate before classified writes** ‚Äî no `classified/*.md` write precedes the 500-char preview + `[Y/n]` checkpoint (S15).
7. **Read-only diagnostics** ‚Äî a `distribute` completeness STOP and a `debug` audit both emit reports with **zero** intended writes (S16, S24).
8. **Local `R` resolution** ‚Äî `R` is resolved by walking up to a domain marker; no absolute/global path is emitted, and an absent marker yields a report, not a write (S8).
